# Study notes

This directory holds short notes on OSS competitors we studied while
implementing roadmap items. One file per repo. Keep each note to ~5
bullets — the goal is "what's worth taking, what's not" not a
re-implementation guide.

## Template

```markdown
# <repo>

- **URL:** https://github.com/<org>/<repo>
- **Commit pinned:** <sha>
- **Studied while working on:** <roadmap item id>

## What it does that we don't

- bullet

## What's clever

- bullet

## What we should take

- bullet

## What we should NOT take

- bullet

## Our improvement angle

- bullet
```

Land your note before opening the implementation PR; the PR body
should link it.
